AutoGate boarding bridge and docking guidance system kit for X-Plane®

Overview

This kit allows X-Plane scenery designers to add animated boarding bridges and docking guidance systems (DGS) to scenery packages. One boarding bridge and two DGSs are included.

The boarding bridge animates to "dock" with the plane's main door when the pilot shuts down the plane's engines with the plane within ½m of the correct stopping position. The (optional) DGS guides the pilot to the correct stopping position.

How it works

The boarding bridge and DGS scenery objects are animated using X-Plane custom datarefs. The values of these datarefs are supplied by a plugin. The plugin needs to know the location of the main boarding door on the plane that the user is piloting and, if the plane is an Airbus or Boeing, the type of the plane.

  • In X-Plane 8.50 and later the plugin reads this information from the "boarding door location" and "ICAO code" values in the ACF file. These values can be edited on the Standard » Viewpoint screen in Plane-Maker.
  • The plugin also contains hard-coded values for a few planes, including the default United 747.

If the plugin is not installed or if the plugin cannot determine the location of the main boarding door then the DGS will stay blank and the boarding bridge will stay in the undocked position.

The system is designed so that it imposes no more load on X-Plane than normal "static" scenery objects unless the user's viewpoint is within 1000m of a boarding bridge or 500m of a DGS, in which case the load remains very modest.

Usage

Copy one of AutoGateLinux.xpl, AutoGateMac.xpl or AutoGateWin.xpl from the plugins folder to your X-Plane/Resources/plugins folder. Please do not re-distribute the plugins with your scenery package; ask your users to download them from http://marginal.org.uk/x-planescenery/AutoGate.zip instead. This ensures that they have the latest version. (Future versions of the plugin may support more bridges and/or DGSs, but will will remain backwards-compatible). That download does not contain any documentation on the operation of the boarding bridge or DGS. You may wish to supply such documentation with your scenery package; you can use this as a starting point.

The boarding bridge object and bitmap file are AutoGate.obj and AutoGate.png in the objs folder. Place as many copies of AutoGate.obj as you like in your scenery:

  • The bridge should be placed with the heading that you want the plane to have when it stops at the gate.
  • Place bridges with their origins more than 20m away from each other. For reference, the distance between the "elbow" and "wrist" is 20m and the distance between the extended centreline and the "elbow" is 10m.

Use of a DGS is optional - the boarding bridge works fine without it - but it will be hard for the pilot to position the plane correctly if you don't include a DGS. Two types of DGS are supplied:

  1. A Safegate® inductive-loop DGS. These were installed at airports up to year 2000.
  2. A Safedock® 2S advanced DGS. These were first installed at airports around year 2000.

Various versions of these DGS objects are supplied in the objs folder, eg Safedock2S-6m.obj. These differ in the height of the DGS above the ground and whether the DGS is mounted on a pole. Each type of DGS uses a normal and a night-time bitmap file.

  • Place at most one DGS at each boarding bridge.
  • The DGS will remain blank or will display incorrect information if it is not placed next to a boarding bridge.
  • For correct operation, the DGS must be placed within the highlighted region behind the bridge object's origin:
  • The Safegate DGS works by line-of-sight, and so must be placed on the extended centreline.

Customisation

You can rename the objects and have multiple versions with different file names. The above rules still apply irrespective of the objects' names.

Boarding bridge

You can change the following things about the boarding bridge without breaking the system. If you do change the boarding bridge object and/or bitmap then I recommend giving your versions a different file name.

If you use Blender then use the file AutoGate.blend in the objs folder as a starting point since importing AutoGate.obj will cause the animation to be lost. If you use another 3D modelling application then check that it can correctly import and re-export AutoGate.obj (including animations) before you spend a lot of time editing.

  • You can re-texture the boarding bridge (if you're using Blender don't forget to also re-texture the version in Layer 2).
  • You can remove or change the centreline quad. However the plugin expects the stopping point to be along the line x=0 and you can't change this behaviour.
  • You can make any changes you like to the "upper arm" and "elbow".
  • You can change the appearance of parts of the boarding bridge below the "elbow" but with the following constraints:
    • Don't move or change the animations (Armature location and pose in Blender) at all.
    • The plugin expects that the floor of the doorway is 4m above the ground when the object is at rest (animation frame 1 in Blender).
    • The plugin expects that that the edge of the doorway is 7.5m west of an unrotated object's origin when the object is at rest (animation frame 1 in Blender).

As supplied, the boarding bridge includes a static reduced level-of-detail version for viewing at distances above 1000m (in Blender this is in Layer 2). You should retain this feature to reduce the load on X-Plane.

Docking Guidance System

The supplied DGSs are not designed to be customisable. However the plugin publishes the following datarefs that you can use to construct your own DGS. Contact the author if you would like additional datarefs.

The supplied DGSs include a static reduced level-of-detail version for viewing at distances above 500m. You should do the same in your DGS to reduce the load on X-Plane.

marginal.org.uk/dgs/status
0 - "Idle". All other published datarefs will also have value 0.
1 - "Active". A plane is approaching the gate and the plane's doorway location is known.
2 - "Stop". The plane is at the stopping point.
3 - "Stopped". The plane has stopped engines at the stopping point.
4 - "Too far". The plane is beyond the stopping point.
 
marginal.org.uk/dgs/track
0 - "Idle/Stop/Too far". Either no plane is approaching the gate or the plane is at or beyond the stopping point (status=0, 2, 3 or 4).
1 - "Active". A plane is approaching the gate and the plane's doorway location is known (status=1).
2 - "Tracking". The plane is >6m and <≈60m from the stopping point (status=1).
3 - "Tracking 6m". The plane is <6m from the stopping point (status=1).
 
marginal.org.uk/dgs/lr
0 - "Idle/Active" (status=0 or track=1).
1 - "Tracking left". The plane is to the left of the stopping point (track=2 or 3).
2 - "Tracking right". The plane is to the right of the stopping point (track=2 or 3).
3 - "Stop/Too far". The plane is at or beyond the stopping point (track=0).
 
marginal.org.uk/dgs/azimuth
0 if track=0 or 1.
Distance (in increments of 0.5m) that the plane is to the right of the stopping point if track=2 or 3. Negative when the plane is to the left of the stopping point. Has value -4 or +4 if the plane is >4m to the left or right of the stopping point.
 
marginal.org.uk/dgs/distance
0 if track=0 or 1.
Distance (in increments of 0.5m) of the plane from the stopping point if track=2 or 3. Has value 12 if the plane is >12m from the stopping point.
 
marginal.org.uk/dgs/distance2
0 if track=0 or 1.
Distance-6m (in increments of 0.5m) of the plane from the stopping point if track=2. Has value 6 if the plane is >12m from the stopping point.
Distance (in increments of 0.5m) of the plane from the stopping point if track=3.
 
marginal.org.uk/dgs/id1
0 if status≠1 or plane is other than:
1 - A300, 2 - A310, 3 - A320, 4 - A330.
 
marginal.org.uk/dgs/id2
0 if status≠1 or plane is other than:
1 - A340, 2 - A350, 3 - A380, 4 - B717.
 
marginal.org.uk/dgs/id3
0 if status≠1 or plane is other than:
1 - B737, 2 - B747, 3 - B757, 4 - B767.
 
marginal.org.uk/dgs/id4
0 if status≠1 or plane is other than:
1 - B777, 2 - B787, 3 - Avro/RJxx, 4 - unknown.

Upgrade information

In version 0.2 of this kit the objects AutoGate.obj and Safedock2S-*.obj all used the same bitmap file - AutoGate.png. In this version AutoGate.png has been changed and is now only used by AutoGate.obj. The Safedock2S-*.obj objects now use the bitmap files Safedock2S.png and Safedock2S_LIT.png.

Therefore if you use the new versions of any of these objects please be sure to also use the new versions of AutoGate.png, Safedock2S.png & Safedock2S_LIT.png.
